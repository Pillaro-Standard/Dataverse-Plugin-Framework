# Early-Bound Generation

Generate strongly-typed early-bound classes from your Dataverse environment metadata using the Power Platform CLI (pac modelbuilder).

> [!NOTE]
> The `Tools` folder is created after rebuilding the project. If it does not appear in Solution Explorer, enable **Show All Files**, locate the generated folder, and choose **Include In Project** so Visual Studio tracks it in the project tree.

## Quick start
1. Install the Power Platform CLI: https://learn.microsoft.com/power-platform/developer/cli/introduction
2. Authenticate: `pac auth create --url https://your-org.crm4.dynamics.com`
3. (Optional) Select profile: `pac auth select --index 1`
4. From project root run: `Tools\EarlyBound\GenerateEarlyBound.bat`

> [!IMPORTANT]
> Do not store connection strings or secrets in committed files.

- Tools/EarlyBound/GenerateEarlyBound.bat — package-managed wrapper

- Tools/EarlyBound/EarlyBoundSettings.json — user-owned settings (never overwritten)

## Settings
EarlyBoundSettings.json controls which entities and messages are generated (entityNamesFilter and messageNamesFilter). Keep messageNamesFilter empty if you do not need messages.

Example to disable messages:

~~~json
{
  "generateSdkMessages": false,
  "messageNamesFilter": []
}
~~~

Example to enable messages:

~~~json
{
  "generateSdkMessages": true,
  "messageNamesFilter": ["new_*"]
}
~~~

Note: messageNamesFilter must not contain an empty string.

## Generated output
Generated C# files are written to `EarlyBound/` in the project root. Classic projects can include these files through the package target. SDK-style projects usually include generated `.cs` files automatically through default compile items.
