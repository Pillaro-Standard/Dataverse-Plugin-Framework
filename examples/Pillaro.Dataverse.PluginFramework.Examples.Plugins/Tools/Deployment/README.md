# Deployment Tools

This folder contains lightweight wrapper scripts that invoke the Pillaro Dataverse CLI.

Important: in this repository the tooling files under Tools/ (wrappers, ILMerge, early-bound helpers) are provided and managed manually in-source, they are not always restored from the NuGet package. The wrapper scripts in this folder are package-managed by default when the framework is consumed from NuGet, but in this repository they are maintained in the examples project and may be adjusted for local development.

Use the PowerShell wrapper in CI; the .bat wrapper is convenient for local Windows runs.

> [!NOTE]
> The `Tools` folder is created after rebuilding the project. If it does not appear in Solution Explorer, enable **Show All Files**, locate the generated folder, and choose **Include In Project** so Visual Studio tracks it in the project tree.

## Quick start
From project root (Windows):

    .\Tools\Deployment\DeployPlugins.bat

Local development (solution):

1. Build the CLI project that ships the pillaro-dv tool:

    dotnet build tools\Pillaro.Dataverse.PluginFramework.Cli -c Debug

2. Run the wrapper (it will try to locate the built pillaro-dv.dll in the CLI project's bin output):

    .\Tools\Deployment\DeployPlugins.bat

> [!IMPORTANT]
> Do not store connection strings or secrets in committed files.

## Local environment
Set the Dataverse connection string in the user environment (PowerShell):

    [Environment]::SetEnvironmentVariable('DV_CONN', 'AuthType=ClientSecret;Url=https://your-org.crm4.dynamics.com;ClientId=your-client-id;ClientSecret=your-client-secret', 'User')

Restart Visual Studio, PowerShell or your terminal after setting the variable.

Temporary session-only setup:

    $env:DV_CONN = 'AuthType=ClientSecret;Url=https://your-org.crm4.dynamics.com;ClientId=your-client-id;ClientSecret=your-client-secret'

Safe verification:

    [Environment]::GetEnvironmentVariable('DV_CONN', 'User') -ne $null
